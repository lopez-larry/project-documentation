# Performance Measurement Plan

## 1. Project Overview
- **Project Name:** Apollo Website Redesign
- **Project Manager:** Sarah Chen
- **Start Date:** 2026-05-01
- **End Date:** 2026-07-31
- **Objective:** Deliver a modern, responsive website with improved UX and SEO
- **Scope Summary:** Design, development, testing, and deployment of new website

---

## 2. Measurement Approach
- **Methodology:** Earned Value Management (EVM) + KPI tracking
- **Tools Used:** Jira, GitHub, Excel Dashboard
- **Reporting Frequency:** Weekly (Sprint Review)
- **Measurement Level:** Task, Sprint, and Phase

---

## 3. Performance Baselines
- **Scope Baseline:** Approved design + feature backlog
- **Schedule Baseline:** 12-week project timeline
- **Cost Baseline:** $119,500

---

## 4. Key Performance Indicators (KPIs)

### 4.1 Schedule Metrics
| Metric | Formula | Description |
|--------|--------|-------------|
| Schedule Variance (SV) | EV - PV | Measures schedule deviation |
| Schedule Performance Index (SPI) | EV / PV | Efficiency of schedule |

---

### 4.2 Cost Metrics
| Metric | Formula | Description |
|--------|--------|-------------|
| Cost Variance (CV) | EV - AC | Measures cost over/under budget |
| Cost Performance Index (CPI) | EV / AC | Cost efficiency |

---

### 4.3 Quality Metrics
| Metric | Description | Target |
|--------|-------------|--------|
| Defect Density | Bugs per feature/module | < 2 per module |
| Test Coverage  | % of code tested | ≥ 85% |

---

### 4.4 Resource Metrics
| Metric | Description | Target |
|--------|-------------|--------|
| Resource Utilization | % of allocated time used | 85%–95% |
| Productivity Rate    | Story points per sprint | ≥ 30 points |

---

## 5. Data Collection Plan
- **Data Sources:** Jira (tasks), GitHub (commits), Timesheets
- **Collection Method:** Automated dashboards + manual validation
- **Validation Process:** Weekly PM review
- **Data Owners:** Team leads (Dev, QA, Design)

---

## 6. Performance Reporting
- **Report Types:**
  - Weekly Status Report
  - Sprint Burndown Chart
  - Executive Summary
- **Audience:**
  - Internal Team (detailed)
  - Stakeholders (summary)
- **Format:** Charts, tables, and narrative summary
- **Frequency:** Weekly

---

## 7. Thresholds & Triggers
| Metric | Threshold | Action Required |
|--------|----------|-----------------|
| CPI    | < 0.9    | Investigate cost overrun |
| SPI    | < 0.9    | Adjust schedule/resources |
| Defect Rate | > 3/module | Increase QA effort |

---

## 8. Variance Analysis
- **Variance Identification Process:**
  - Compare EV vs PV vs AC weekly
- **Root Cause Analysis Method:**
  - Identify delays (design, dev, QA)
- **Corrective Actions:**
  - Reallocate resources
  - Adjust sprint scope
- **Preventive Actions:**
  - Improve backlog grooming
  - Increase test automation

---

## 9. Change & Escalation Process
- **Escalation Path:** Project Manager → Sponsor → Steering Committee
- **Approval Authority:** Project Sponsor
- **Decision Timeline:** 48 hours for critical issues

---

## 10. Continuous Improvement
- **Lessons Learned Process:** End of each sprint retrospective
- **Feedback Loops:** Sprint reviews + stakeholder demos
- **Process Adjustments:** Update workflow based on retrospectives
